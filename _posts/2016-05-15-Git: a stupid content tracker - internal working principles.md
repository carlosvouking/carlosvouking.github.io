[Wikipedia](https://en.wikipedia.org/wiki/Git) says that **'Git is a distributed revision control system'**. It tracks files and directories accross computer systems(_distribution_). Tracking and interacting with those files and directories require the use of specific commands in a bash terminal.

Git appears to be like a mapping table which links **keys** and **values** and save the (key, values) pairs in a system. To make it simple, let's say I am writing my thesis and as it goes, I would like to gradually edit it and save every edited copy of it. Each edited copy is mapped to a specific unique key and stored in a table. Git is just that table. Technically, the text in my thesis report is called a **body** and represents a sequence of bites.
